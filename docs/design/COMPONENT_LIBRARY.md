## Component Rules

Каждый компонент должен иметь:

- назначение;
- typed props;
- states;
- variants, если нужны;
- accessibility behavior;
- keyboard support;
- ограничения.

Shared UI компоненты не знают о Project, Product, Pipeline lifecycle и Orchestrator decisions.
